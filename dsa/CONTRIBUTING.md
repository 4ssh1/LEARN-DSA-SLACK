# Contributing to DSA Roadmap 2025

Thank you for your interest in contributing! This document provides guidelines and instructions for contributing.

## How to Contribute

### 1. Submit Solutions
- Add your solutions to the appropriate week's directory
- Follow the naming conventions for your language
- Ensure your code includes comments and is well-structured
- Test your solution before submitting

### 2. Share Community Solutions
- Place alternative approaches in `community-solutions/`
- Include your GitHub username in the filename
- Add a brief explanation of your approach

### 3. Improve Documentation
- Fix typos or improve clarity
- Add examples to explanations
- Create visual diagrams for complex concepts

### 4. Report Issues
- Use GitHub Issues for bug reports
- Include clear reproduction steps
- Provide expected vs. actual behavior

## Guidelines

### Code Quality
- Write clean, readable code
- Use meaningful variable names
- Add comments for complex logic
- Follow language conventions

### Naming Conventions
- Python: `snake_case.py`
- JavaScript: `camelCase.js`
- C++: `snake_case.cpp`
- C#: `PascalCase.cs`

### Pull Request Process
1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test thoroughly
5. Submit a pull request with description

## Code of Conduct

Please be respectful and inclusive. See [CODE_OF_CONDUCT.md](.github/CODE_OF_CONDUCT.md) for details.

Thank you for contributing!
